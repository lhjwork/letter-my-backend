# 📮 Letter Community 프로젝트 기반 자기소개서

## 🎯 프로젝트 소개

안녕하세요. 저는 **디지털과 아날로그를 연결하는 혁신적인 서비스**를 개발한 백엔드 개발자입니다.

**Letter Community**는 단순한 편지 공유 플랫폼을 넘어, **디지털 편지를 실제 종이 편지로 배송하는 서비스**를 핵심으로 하는 소셜 플랫폼입니다. 이 프로젝트를 통해 복잡한 비즈니스 로직 구현, 확장 가능한 아키텍처 설계, 그리고 사용자 중심의 서비스 개발 역량을 입증했습니다.

## 🚀 핵심 성과 및 기술적 도전

### 1. 혁신적인 비즈니스 로직 구현 ⭐

**"디지털을 아날로그로 연결하는" 3가지 실물 편지 시스템을 설계하고 구현했습니다:**

- **기본 실물 편지**: 편지당 1회 신청 제한, 즉시 처리
- **누적 실물 편지**: 여러 명이 동일 편지 신청, 일정 수량 달성 시 일괄 처리
- **작성자 승인 시스템**: 무제한 신청 가능, 작성자가 개별 승인/거절

이 중에서도 **작성자 승인 시스템**은 가장 복잡한 비즈니스 로직으로, 익명 사용자도 세션 기반으로 여러 번 신청할 수 있으면서도 스팸을 방지하고, 작성자가 각 신청을 개별적으로 관리할 수 있는 시스템입니다.

```typescript
// 핵심 구현 예시: 세션 기반 익명 사용자 신청 관리
const sessionId = req.sessionID;
const existingRequests = await AuthorApprovalPhysicalRequest.countDocuments({
  letterId,
  sessionId,
  status: { $ne: "cancelled" },
});

if (existingRequests >= MAX_REQUESTS_PER_SESSION) {
  return res.status(400).json({
    success: false,
    message: `세션당 최대 ${MAX_REQUESTS_PER_SESSION}개까지만 신청 가능합니다.`,
  });
}
```

### 3. 복잡한 문제 해결 능력

#### ES Module 호환성 문제 해결

**문제**: `isomorphic-dompurify`와 `jsdom` 간 ES Module 충돌로 HTML 정제 기능 오류
**해결**: cheerio 기반 순수 HTML 정제 로직을 직접 구현

- XSS 방어 기능 유지하면서 성능 개선
- 타입 안전성 확보

```typescript
// 직접 구현한 HTML 정제 함수
export function sanitizeHtml(html: string): string {
  const $ = cheerio.load(html);

  // 허용된 태그만 유지
  const allowedTags = ["p", "br", "strong", "em", "u", "s"];
  $("*").each((_, element) => {
    if (!allowedTags.includes(element.tagName)) {
      $(element).replaceWith($(element).text());
    }
  });

  return $.html();
}
```

#### MongoDB Atlas → Local 마이그레이션

**문제**: 클라우드 DB에서 로컬 개발 환경으로 데이터 이전 필요
**해결**: 자동화된 마이그레이션 스크립트 개발

- 3개 컬렉션 (users: 3, letters: 13, tests: 1) 성공적 이전
- 데이터 무결성 검증 로직 포함

#### Git 보안 이슈 해결

**문제**: `.env.production` 파일이 실수로 커밋되어 MongoDB 자격증명 노출
**해결**: `git filter-branch`로 히스토리에서 완전 제거

- 보안 정책 강화 및 `.gitignore` 업데이트
